## 状態

- Phase: 1
- Status: done

## 目的

月次レポートのヒートマップを、Codex App のプロフィールヒートマップに近いカレンダー型表示へ改善する。

## 要件

- 横軸は日付にする
- 横軸は直近30日の各日付を1日1列で表示する
- 横軸は各列に `MM/DD` を表示し、連結された日付数字は表示しない
- 該当しない曜日の交点は `.` で表示し、該当する曜日の交点だけ `■` を表示する
- 縦軸は曜日にする
- 曜日は下から日、月、火、水、木、金、土の順にする
- セルは Unicode block 文字で表示する
- `##` と `--` は使わない
- 色分けは既存の正解率ベースの段階を維持する
- 統計保存形式と集計 API は変更しない

## 実装予定

- `src/reports.rs` の `create_heatmap_without_badges` を日付横軸・曜日縦軸の描画へ変更する
- セル表示と凡例の block 文字を統一する
- 表示幅が不足するケースでも日付と曜日の対応が崩れないようにする
- 必要に応じてヒートマップ描画のユニットテストを追加する

## 完了条件

- 下から日、月、火、水、木、金、土の順で表示される
- `##` と `--` が表示されない
