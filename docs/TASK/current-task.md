# Current Task: レポートヒートマップ密度調整

## 状態

- Phase: 1
- Status: done
- Created: 2026-07-02 09:36

## 目的

月次レポートのヒートマップから週列ヘッダーを削除し、`■` と `·` を Codex のプロフィールヒートマップに近い密度で詰めて表示する。

## 要件

- 週列ヘッダーは表示しない
- 縦軸の曜日ラベルは維持する
- 曜日は下から日、月、火、水、木、金、土の順にする
- 描画時の行順は上から土、金、木、水、火、月、日とする
- 横方向のセルは左から古い週、右へ新しい週の順にする
- 対象期間外、またはその週に存在しない曜日のセルは `·` で表示する
- 対象期間内の日は Unicode block 文字で表示する
- セル同士の間隔は最小限に詰める
- `##` と `--` は使わない
- 色分けは既存の正解率ベースの段階を維持する
- 統計保存形式と集計 API は変更しない

## 実装予定

- `src/reports.rs` のヒートマップヘッダー描画を削除する
- セル幅を詰める定数に変更する
- 対象期間外セルと対象期間内セルの描画を新しい幅に合わせる
- ヒートマップ描画のユニットテストをヘッダーなし前提へ更新する
