# PLAN

## 目的

レポート画面の月次ヒートマップを、縦軸を曜日、横軸を週、セルを日にした表示へ変更する。

## 作業方針

- `src/reports.rs` の月次ヒートマップ描画に変更範囲を限定する
- 統計データの保存形式と日次集計 API は変更しない
- 縦軸は曜日にする
- 曜日行は上から土、金、木、水、火、月、日とし、下から日、月、火、水、木、金、土に積む
- 横軸は対象期間を週単位に分割し、左から古い週、右へ新しい週を表示する
- 週列ヘッダーは表示しない
- 各セルは該当する日の結果を表す
- 対象期間外、またはその週に存在しない曜日のセルは `·` で表示する
- セル同士の間隔は最小限に詰める
- 対象期間内の日は Unicode block を使い、`##` や `--` のような ASCII 代替表現は使わない
- 色分けは既存の正解率ベースの色を維持する
- 実装後に `make check`、`make build`、コードレビューを通す

## Phase 1: レポートヒートマップ改善

### 目的

レポート画面の月次ヒートマップを、縦軸を曜日、横軸を週、セルを日にしたカレンダー型の表示へ変更する。

- 横軸は週にする
- 週列ヘッダーは表示しない
- 各セルは該当する日を表す
- 週内に該当日がないセルは `·` で表示する
- セル同士の間隔は最小限に詰める
- セル表示は Unicode block を使い、`##` や `--` のような ASCII 代替表現は使わない

## Phase 2: 成績レポート対象期間の90日化

### 目的

成績レポートで表示・集計する対象期間を、今日を含む直近30日から直近90日に変更する。

### 作業方針

- 統計データの保存形式は変更しない
- 既存の集計 API は日数を引数で受け取る設計を維持する
- レポート画面から渡す対象日数を90日に変更する
- 画面上の「30日」「月次」「1ヶ月」表記を90日相当に更新する
- ヒートマップの週横軸・曜日縦軸の表示方針は維持する
- 90日表示でも端末幅が不足する場合は、既存の固定幅計算と切り詰めの責務内で扱う
- 実装後に `make check`、`make build`、コードレビューを通す
