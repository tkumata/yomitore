# PLAN

## 目的

成績レポートで表示・集計する対象期間を、今日を含む直近90日から直近180日に変更する。

## 作業方針

- 統計データの保存形式は変更しない
- 既存の集計 API は日数を引数で受け取る設計を維持する
- レポート画面から渡す対象日数を180日に変更する
- 評価スコア集計と日次ヒートマップは同じ対象日数を使う
- 画面上の「90日」表記を180日相当に更新する
- ヒートマップの週横軸・曜日縦軸の表示方針は維持する
- 180日表示でも端末幅が不足する場合は、既存の固定幅計算と切り詰めの責務内で扱う
- 実装後に `make check`、`make build`、コードレビューを通す

## Phase 1: 成績レポート対象期間の180日化

### 目的

成績レポートの保存・表示対象期間を、今日を含む直近180日に変更する。

### 作業方針

- `src/reports.rs` のレポート対象期間定数を180日に変更する
- 評価スコアタイトルとヒートマップタイトルの表示文言を更新する
- 180日対象のヒートマップ計算に合わせて既存ユニットテストの期待値を維持できるか確認する
- README と DOCUMENT の現在事実を180日相当に更新する
- 統計保存形式と集計 API は変更しない
- 実装後に `make check`、`make build`、コードレビューを通す
