# ADR: 月次ヒートマップを週横軸と曜日縦軸にする

## 状況

現在の月次ヒートマップは、直近30日を日付列として横に並べ、曜日を縦軸にしている。この表示は日付の連続性を追いやすい一方で、横幅を大きく使い、週単位のまとまりを把握しづらい。

ユーザーは、縦軸を曜日、横軸を週、セルを日にした表示を求めている。表示例では、上から土、金、木、水、火、月、日の行を並べ、各週列の該当日を `■`、該当しないセルを `.` で表している。

## 決定

月次ヒートマップの描画を、週を横軸、曜日を縦軸、セルを日にしたカレンダー型表示へ変更する。

- 対象期間は今日を含む直近30日とする
- 横軸は週とし、左から古い週、右に新しい週を置く
- 週は日曜始まりで扱う
- 曜日は下から日、月、火、水、木、金、土の順にする
- 描画時の行順は上から土、金、木、水、火、月、日とする
- 対象期間外、または週内に存在しない曜日のセルは `.` で表示する
- 対象期間内の日は Unicode block で表示する
- 色分けは既存の正解率ベースの段階を維持する
- 統計データ構造、保存形式、集計 API は変更しない

## 理由

週を横軸にすると、直近30日の表示幅を抑えながら、カレンダーとしての週単位のまとまりを維持できる。曜日を縦軸にすることで、特定曜日の学習傾向も引き続き確認できる。

対象期間外セルを `.` として残すことで、週列と曜日行の対応が崩れず、月初・月末に相当する不完全な週も読みやすくなる。

## 結果

実装フェーズでは `src/reports.rs` の月次ヒートマップ描画を中心に変更する。完了判定は `make check`、`make build`、コードレビューの成功とする。
