# ADR: 月次ヒートマップの週ヘッダーをなくしてセルを詰める

## 状況

現在の月次ヒートマップは、横軸を週、縦軸を曜日にしているが、週列ヘッダーと広いセル幅により表示が横に間延びしている。

ユーザーは Codex のプロフィールヒートマップのように、ヘッダーを持たず、`■` と `·` を詰めた密度の高い表示を求めている。

## 決定

月次ヒートマップでは週列ヘッダーを表示しない。曜日ラベルの右側に、週単位のセルを最小限の間隔で横に並べる。

- 縦軸の曜日ラベルは維持する
- 横軸の週列ヘッダーは表示しない
- 対象期間内の日は `■` で表示する
- 対象期間外、または週内に存在しない曜日のセルは `·` で表示する
- セル同士の間隔は最小限に詰める
- 色分けは既存の正解率ベースの段階を維持する
- 統計データ構造、保存形式、集計 API は変更しない

## 理由

週列ヘッダーを削除し、セル幅を詰めることで、ヒートマップを視覚的な活動パターンとして読みやすくする。週の正確な日付よりも、曜日別・週別の密度を把握することを優先する。

## 結果

実装フェーズでは `src/reports.rs` の月次ヒートマップ描画と関連テストを更新する。完了判定は `make check`、`make build`、コードレビューの成功とする。
