# ADR: ハーネスにレビューゲートを追加する

## 状況

`check` と `build` が通っても、変更が妥当とは限らない。ハーネスが `build` 成功時点で終了すると、差分確認や設計逸脱の検出が抜ける。

## 決定

- `build` 成功後は即終了せず `review_pending` に遷移する
- `Stop` / `agentStop` ではレビュー未完了なら完了扱いにしない
- `review_pipeline.sh` を手動実行する入口として追加し、承認時のみ `done` にする
- `review_pipeline.sh` 承認時にワークツリーのスナップショットを保存し、差分が変わった場合だけ `check_pending` に戻す

## 理由

- 検証とレビューは目的が異なる
- `build` 成功は実行可能性の確認であり、レビューは変更妥当性の確認である
- 状態を分けることで、レビュー漏れをハーネス側で防げる

## 結果

- `check -> build -> review -> done` の流れになった
- レビュー未完了のまま作業終了しづらくなった
- 差し戻しのときは `review_pending` を維持して再レビューできる
- 完了後に同じ未コミット差分で無限に再点火しない
- 完了後に差分が変わったときだけ再点火できる
