# ADR: 監査で確認した過剰設計を削除する

## 背景

全体監査により、単一実装の API トレイト、ビルド時コード生成、階層化された状態フラグ、未使用ファイルと依存関係、複雑な検証状態管理が確認された。

## 決定

- `ApiClient` は具象型として直接使用し、評価解析は純粋関数でテストする
- `docs/HELP.md` は Rust 標準の `include_str!` で埋め込む
- 重複しない状態フラグだけを `App` に直接保持し、入力状態は既存の `TextAreaState::focus` を使う
- 未使用ファイル、完了済み TODO、未使用依存を削除する
- 検証フックは単一フィンガープリントと check/build の2段階だけを保持する
- 保存データ形式とユーザー向け機能は変更しない

## 結果

- 保守対象となる型、ファイル、依存関係、状態遷移が減る
- 外部 API を使わない評価解析テストは維持される
- Rust 関連差分だけが検証フックの対象となる
