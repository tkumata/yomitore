# ADR: 月次ヒートマップを日付横軸と曜日縦軸にする

## 状況

現在の月次ヒートマップは、週を行、曜日を列として表示している。この表示はコンパクトだが、日付の連続性や曜日ごとの傾向を直感的に追いづらい。

ユーザーは Codex App のプロフィールヒートマップに近い表示を望んでいる。具体的には、横軸を日、縦軸を曜日にし、下から日、月、火、水、木、金、土の順に並べる。セルは `##` や `--` ではなく、btop のメーターに近い Unicode block で表示する。

## 決定

月次ヒートマップの描画を、日付を横軸、曜日を縦軸にしたカレンダー型表示へ変更する。

- 対象期間は今日を含む直近30日とする
- 日付は左から古い日、右に新しい日を置く
- 曜日は下から日、月、火、水、木、金、土の順にする
- 描画時の行順は上から土、金、木、水、火、月、日とする
- セルには Unicode block を使う
- 色分けは既存の正解率ベースの段階を維持する
- 統計データ構造、保存形式、集計 API は変更しない

## 理由

日付を横軸にすると、直近30日の流れを左から右へ自然に読める。曜日を縦軸にすると、特定曜日の学習傾向も見つけやすい。

Unicode block を使うことで、既存の色分けを活かしつつ、ASCII 代替表現より密度の高い見た目にできる。データ構造を変えないため、既存統計ファイルとの互換性も維持できる。

## 結果

実装フェーズでは `src/reports.rs` の月次ヒートマップ描画を中心に変更する。完了判定は `make check`、`make build`、コードレビューの成功とする。
