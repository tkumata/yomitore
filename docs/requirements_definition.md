# 要件定義書: 読解力トレーニング CLI (yomitore)

## 1. 概要

### 1.1. プロジェクトの目的

本プロジェクトは、CloudGroq API を活用した対話型の CLI アプリケーションを開発することを目的とする。ユーザーは、AI が生成した堅い文章を要約するトレーニングを通じて、読解力および要約能力を向上させることができる。

### 1.2. ターゲットユーザー

- 文章の読解や要約に苦手意識を持つ学生や社会人
- 簡潔で的確な文章作成能力を鍛えたい全ての個人

### 1.3. 開発言語

- Rust

## 2. 機能要件

### 2.1. 認証機能

- アプリケーション起動時に、CloudGroq API キーの入力を求めるプロンプトを表示する。
- 入力された API キーの有効性を検証する。
- 認証に成功するまで、後続の処理に進むことはできない。認証失敗時はエラーメッセージを表示し、再入力を促す。

### 2.2. 文章生成機能

- 認証成功後、CloudGroq API に対し「日本語の公的文書のようなお堅い文章を 200 文字程度で生成してください。」という指示を送信する。
- API から返却された文章をターミナル上に表示する。

### 2.3. 要約入力機能

- 文章表示後、ユーザーに対してその文章の要約を入力するよう促すプロンプトを表示する。
- ユーザーによる複数行のテキスト入力を受け付ける。
- `Shift + Enter` キーの組み合わせによって、入力の完了を検知する。

### 2.4. 要約評価機能

- ユーザーの入力完了後、以下の 2 つの情報を CloudGroq API に送信する。
  - **原文**: アプリケーションが生成した文章
  - **要約文**: ユーザーが入力した文章
- API に対し、要約文が原文の内容を適切に要約できているかを評価させる。

### 2.5. 結果表示機能

- API から返された評価結果（例：「要約は適切です」「改善の余地があります」など）をターミナル上に表示する。

### 2.6. 繰り返し実行機能

- 評価結果を表示した後、任意のキー入力を待つ状態になる。
- ユーザーが何らかのキーを入力すると、次のトレーニングサイクル（2.2. 文章生成機能から）を開始する。

## 3. 非機能要件

### 3.1. 実行環境

- ターミナル上で動作するクロスプラットフォーム（Windows, macOS, Linux）の CLI アプリケーションであること。

### 3.2. ユーザーインターフェース

- 全ての操作は、標準入出力を通じてテキストベースで行われる。

### 3.3. セキュリティ

- 入力された API キーは、ターミナル上に表示されることなく、安全に取り扱う。

### 3.4. エラーハンドリング

- API 通信の失敗（ネットワークエラー、無効な API キーなど）が発生した場合、アプリケーションがクラッシュすることなく、適切なエラーメッセージを表示する。

### 3.5. 終了方法

- `Ctrl+C` のシグナルを検知した場合、アプリケーションは適切に終了する。
